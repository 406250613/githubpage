### 自动更新完整流程

```
开发者修改代码并部署
       ↓
浏览器检测到 service-worker.js 字节变化（每次打开页面或每24小时轮询）
       ↓
新 SW 在后台静默安装 + 缓存新资源
       ↓
install 完成 → skipWaiting() → 新 SW 立即激活
       ↓
activate → clients.claim() → 接管所有标签页
       ↓
页面收到 controllerchange 事件
       ↓
Toast 提示"🔄 检测到新版本，正在更新..."（1.5秒）
       ↓
window.location.reload() → 拿到最新 HTML（Network-First 保证）
```

### 各类文件的更新方式

| 文件类型 | 策略 | 何时更新 |
|---------|------|---------|
| service-worker.js | 浏览器自动检测字节差异 | 每次打开页面 / 24h 轮询 |
| HTML 页面 | Network-First + reload | SW 更新时自动刷新 |
| 图标 / manifest | Cache-First | 修改 `CACHE_NAME` 版本号触发 |

> **注意**：localStorage 数据不受影响，刷新不会丢失任何记录。
